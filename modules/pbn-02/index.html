VEYRA PBN PATH · MÓDULO 02

Performance y errores de navegación

Aprende a distinguir la trayectoria deseada, la posición estimada y la posición real, y comprende cómo los errores parciales se combinan en el error total del sistema.

Duración estimada: 15–18 minutos. Incluye requisitos de performance, modelo TSE, aplicación operacional y evaluación final.
01 · REQUISITOS DE PERFORMANCE

La precisión no trabaja sola

Una operación PBN no se define únicamente por cuántas millas puede desviarse la aeronave. La especificación reúne varios atributos de performance y las funciones necesarias para la operación.

PrecisiónGrado de conformidad entre la posición estimada o trayectoria y la requerida.
IntegridadConfianza en la información y capacidad de alertar cuando no debe utilizarse.
ContinuidadCapacidad de mantener el servicio previsto sin interrupciones no programadas.
DisponibilidadProbabilidad de que el sistema y la infraestructura puedan usarse cuando se necesitan.
FuncionalidadCapacidades requeridas para ejecutar, gestionar y presentar la operación.
Idea clave: cumplir la precisión lateral no basta si el sistema ha perdido integridad, no ofrece la función requerida o la tripulación no puede detectar una degradación.
02 · EL MODELO DEL ERROR

TSE: el resultado total

El Total System Error expresa la diferencia lateral entre la posición real de la aeronave y la trayectoria deseada. Para comprender su origen se analizan tres componentes.

Path Definition Error

Diferencia entre la trayectoria que debería definirse y la trayectoria que el sistema ha definido. Puede relacionarse con datos o definición del procedimiento.

Flight Technical Error

Diferencia entre la trayectoria definida y la forma en que la aeronave realmente la vuela, manualmente o con automatización.

Navigation System Error

Diferencia entre la posición real y la posición que el sistema de navegación estima y presenta.

TSE depende de PDE, FTE y NSELos componentes se consideran estadísticamente; no deben sumarse de forma aritmética en cada instante.
03 · TRES REFERENCIAS

No confundir posición estimada con posición real

REPRESENTACIÓN CONCEPTUAL
Trayectoria deseadaDestino

El sistema guía con base en la posición que estima. La aeronave, sin embargo, ocupa una posición física que puede diferir de esa estimación.

LECTURA CORRECTA
1
Trayectoria deseada
Dónde exige el procedimiento que esté la aeronave.
2
Posición estimada
Dónde calcula el sistema que está la aeronave.
3
Posición real
Dónde se encuentra físicamente la aeronave.
Consecuencia operacional: una indicación centrada no garantiza por sí sola que la posición real coincida exactamente con la trayectoria deseada.
04 · EL CRITERIO DEL 95%

El número de la NavSpec expresa precisión sostenida

En una especificación como RNAV 1 o RNP 1, la aeronave debe mantener el TSE lateral dentro de ±1 NM al menos durante el 95% del tiempo de vuelo aplicable.

DENTRO DEL REQUISITO

La mayor parte de la operación

El criterio estadístico del 95% permite describir una performance sostenida, no una desviación instantánea aislada.

± 1 NM≥ 95% del tiempo
NO SIGNIFICA

Un corredor autorizado para desviarse

No es una invitación a utilizar deliberadamente todo el margen ni reemplaza los límites operacionales, alertas o procedimientos de la tripulación.

No es tolerancia de pilotaje
La NavSpec describe una capacidad del sistema completo, no solamente la precisión del sensor.
05 · APLICACIÓN OPERACIONAL

¿Qué puede aumentar cada error?

PDE · DEFINICIÓN

Datos y procedimiento

Selección incorrecta, procedimiento no vigente, base de datos inadecuada o una modificación no autorizada pueden comprometer la trayectoria definida.

FTE · CONDUCCIÓN

Control y automatización

Técnica de vuelo, anticipación, viento, modo incorrecto o gestión deficiente de la automatización pueden separar a la aeronave de la trayectoria.

NSE · POSICIÓN

Sensores y geometría

Disponibilidad, geometría, interferencia o degradación de sensores pueden afectar la solución de posición estimada.

Defensa de la tripulación: verificar procedimiento y vigencia, confirmar modos y secuencia, comparar la carta con el sistema y responder oportunamente a alertas o degradaciones.
